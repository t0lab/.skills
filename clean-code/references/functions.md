# Clean Functions — Detailed Reference (F1–F4)

## F1: Too Many Arguments (Maximum 3)

```python
# Bad — too many parameters
def create_user(name, email, age, country, timezone, language, newsletter):
    ...

# Good — use a dataclass
@dataclass
class UserData:
    name: str
    email: str
    age: int
    country: str
    timezone: str
    language: str
    newsletter: bool

def create_user(data: UserData):
    ...
```

```typescript
// Bad
function createUser(name: string, email: string, age: number, country: string, tz: string) { ... }

// Good — use an options object
interface CreateUserOptions {
  name: string;
  email: string;
  age: number;
  country: string;
  timezone: string;
}

function createUser(options: CreateUserOptions) { ... }
```

More than 3 arguments is a strong signal the function does too much or needs a data structure.

## F2: No Output Arguments

Don't modify arguments as side effects. Return values instead.

```python
# Bad — modifies argument
def append_footer(report: Report) -> None:
    report.append("\n---\nGenerated by System")

# Good — returns new value
def with_footer(report: Report) -> Report:
    return report + "\n---\nGenerated by System"
```

```javascript
// Bad — mutates input array
function addDefaults(config) {
  config.timeout = config.timeout || 3000;
}

// Good — returns new object
function withDefaults(config) {
  return { timeout: 3000, ...config };
}
```

## F3: No Flag Arguments

Boolean flags mean the function does at least two things. Split it.

```python
# Bad — function does two different things
def render(is_test: bool):
    if is_test:
        render_test_page()
    else:
        render_production_page()

# Good — split into two functions
def render_test_page(): ...
def render_production_page(): ...
```

```typescript
// Bad
function getUsers(includeInactive: boolean) { ... }

// Good
function getActiveUsers() { ... }
function getAllUsers() { ... }
```

## F4: Delete Dead Functions

If a function is not called anywhere, delete it. No "just in case" code. Version control preserves history — you can always get it back.

## G30: Functions Should Do One Thing

This is the most important rule about functions. If you can extract another function from it with a meaningful name that isn't just a restatement of its implementation, the function does more than one thing.

```python
# Bad — does three things
def process_order(order):
    # validate
    if not order.items:
        raise ValueError("Empty order")
    # calculate
    total = sum(item.price * item.qty for item in order.items)
    tax = total * 0.0825
    # save
    db.save(Order(total=total + tax, items=order.items))

# Good — each function does one thing
def validate_order(order):
    if not order.items:
        raise ValueError("Empty order")

def calculate_order_total(order) -> float:
    subtotal = sum(item.price * item.qty for item in order.items)
    return subtotal + calculate_tax(subtotal)

def save_order(order, total):
    db.save(Order(total=total, items=order.items))
```
